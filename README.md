# База данных и словарь данных

## Описание  
В рамках проекта мы перешли от требований к структуре данных. Вместе разобрались, какие сущности есть в системе, какие у них атрибуты и как они связаны между собой. Построили логическую модель БД и диаграмму классов.

## Полученные навыки  
- Выделение сущностей из требований (US, UC)  
- Описание атрибутов (типы, ключи, обязательность)  
- Работа со справочниками и связями  
- Понимание жизненного цикла объектов (статусы)  
- Построение UML-диаграммы классов  

## Этапы  
1. Описание базовой сущности «Персоны»  для сущностей "Клиенты" и "Сотрудники"
2. Описание других сущностей
3. Описание справочников: роли, услуги, услуги мастера
4. Построение диаграмму классов и ее обновление после добавление дополнительных сущностей

## Заключение  
Этот групповой проект помог понять, как из требований получается структура базы данных. Мы научились правильно орагнизовывать данные и устанавливать связи между ними.
